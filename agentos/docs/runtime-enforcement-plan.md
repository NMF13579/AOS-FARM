## Назначение

На данный момент (Build Steps 1-10) вся архитектура AOS-1 базируется на декларативных правилах (контрактах) и скриптах статического анализа (Thin Validator). Это `Policy-Based Governance`.

Данный документ описывает план перехода к `Runtime Enforcement` — механизмам, которые физически блокируют исполнение запрещенных операций в реальном времени. Внедрение этих механизмов запланировано на будущие итерации.

## 1. Command Allowlist (Белый список команд)

Вместо того чтобы разрешать агенту выполнять любые bash/shell команды, будет введен строгий перечень допустимых команд:
- **Разрешено:** `git status`, `git diff`, `npm run test` (только для чтения или безопасного запуска).
- **Запрещено:** `rm -rf`, бесконтрольный запуск `curl/wget` в защищенных зонах.
Реализация: Использование кастомной shell-обертки или перехватчика внутри IDE/Agent-фреймворка.

## 2. Write Allowlist (Белый список на запись)

Агент сможет изменять только файлы, явно перечисленные в `Allowed Changes` в `Task Brief`. 
Любая попытка вызова инструмента `write_to_file` вне этого списка будет отклонена на уровне API агента (Sandbox Execution).

## 3. Protected Path Guard (Защита корневых файлов)

Директория `agentos/docs/`, скрипты `agentos/scripts/` и корневая конфигурация репозитория будут отмечены как `Protected`. 
Даже если агент каким-то образом попытается отредактировать их, файловая система (или хуки) заблокирует изменение, если задача не имеет статуса `HIGH_RISK_PROTECTED` и явной подписи `Approval` от человека.

## 4. Commit / Push Guard (Блокировка коммитов)

Агенту будет запрещено выполнять `git commit` и `git push` напрямую в `main`/`master` или другие protected ветки. 
Реализация: Внедрение `pre-commit` и `pre-push` хуков, которые проверяют наличие `Evidence Report` и файла `Human Review` перед тем, как позволить транзакцию в git.

## 5. Audit Log (Журнал аудита)

Все команды (run_command), выполненные агентом, а также все попытки доступа будут записываться в нередактируемый (append-only) журнал: `agentos/audit/runtime.log`.
Это обеспечит полную прозрачность действий агента и позволит человеку быстро провести расследование в случае нарушений (Dogfood failures).
