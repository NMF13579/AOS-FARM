# SELF_TEST Functional Verification Concept

This concept bridges the Task Quality Package (AOS-FARM.454) with a human-readable checklist.

It explains:
- what is verified by code (task quality validator)
- what requires manual human review
- items left NOT_RUN or UNKNOWN
- Evidence context
- why human review remains strictly required
- why PASS / Evidence / SELF_TEST do not grant approval.
