# Next Task Selection Examples

These examples show review-only selection artifacts that follow a `Next Task
Candidate`.

They are not executable tasks, not Task Briefs, and not execution
authorization.

## Included Examples

- `selection-accept-example.md`
- `selection-clarify-example.md`
- `selection-defer-example.md`
- `selection-reject-example.md`

## Safety Notes

- `ACCEPT` means candidate accepted for Task Brief draft preparation only.
- `ACCEPT` does not authorize execution.
- `ACCEPT` does not authorize commit or push.
- `ACCEPT` does not start the next task.
- PASS is not approval.
- Evidence is not approval.
- UNKNOWN is not OK.
- NOT_RUN is not PASS.
- Human approval cannot be simulated.
