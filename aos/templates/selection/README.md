# Next Task Selection Templates

Use these templates after a `Next Task Candidate` exists and before any Task
Brief draft preparation begins.

## Included Template

- `next-task-selection-decision-template.md` records one human selection
  decision for one candidate.

## Boundary Notes

- `ACCEPT` means candidate accepted for Task Brief draft preparation only.
- `ACCEPT` is not task approval.
- `ACCEPT` does not assign Risk Profile.
- `ACCEPT` does not authorize execution.
- `ACCEPT` does not authorize commit or push.
- `ACCEPT` does not start the next task.
- `CLARIFY` requires more human or user clarification.
- `DEFER` postpones the candidate.
- `REJECT` rejects the candidate and it must not be compiled.
- `REPLACE` requests a different candidate.
- PASS is not approval.
- Evidence is not approval.
- UNKNOWN is not OK.
- NOT_RUN is not PASS.
- Human approval cannot be simulated.
