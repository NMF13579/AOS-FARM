# Первое знакомство с AOS-FARM

## Первый запуск в проекте
Узнайте, с чего начать работу, какие файлы читать первыми и как безопасно проверить статус, в руководстве: [AOS First Start Guide](FIRST-START.md). Также доступно обучение: [Tutor](TUTOR.md) и [Практические сценарии](TUTOR-SCENARIOS.md).

## Что такое AOS?
AOS — это система правил, которая делает работу с AI-агентом безопасной. Она контролирует, чтобы агент не сделал ничего скрытого, опасного или не авторизованного вами.

## Что такое AOS-FARM?
AOS-FARM — это конкретная реализация этих правил и структура директорий, которая организует всю вашу документацию, требования и код так, чтобы AI агент их правильно понимал и безопасно с ними работал.

## Как начать, если вы не программист?
Вам не нужно уметь программировать, чтобы использовать AOS-FARM. Начните с:
1. Прочтения [инструкции по установке](INSTALL.md).
2. Напишите агенту, чтобы он запустил сухую установку (dry-run), когда она будет доступна.
3. Ваши требования и идеи мы будем оформлять в виде понятных документов.
4. Для быстрой проверки состояния используйте [AOS Doctor](../scripts/aos_doctor.py).
5. Для просмотра очереди задач: [Queue Dashboard](../scripts/aos_queue_dashboard.py).

## Где что хранится?
* `/aos/` — само ядро AOS. Сюда нельзя ничего писать.
* `/aos/root/` — шаблоны корневых файлов для установки.
* `/aos/prompt-packs/` — инструкции (Prompt Packs) для разных агентов.
* `/aos-modules/` — контейнер для дополнительных модулей.
* `/project/` — здесь будет храниться вся **документация по вашему проекту** (идеи, ТЗ, архитектура).
* `/.aos-tmp/` — временная папка. В ней нет важных данных, она нужна для рабочих логов.
* **Ваш программный код** хранится в корне вашего репозитория (как обычно), не в папке `/project/` и не в `/aos/`.

## Куда класть файлы
Для понимания того, где хранить документацию, шаблоны и модули, обязательно ознакомьтесь с [Моделью границ рабочего пространства (Workspace Boundary)](WORKSPACE-BOUNDARY.md).

## Важные правила безопасности
* **PASS — это не одобрение.** Если агент говорит, что тесты прошли успешно (PASS), это не значит, что код можно пушить в проект. Окончательное разрешение всегда за вами.
* **Self-test PASS не разрешает выполнение задач.** Самопроверка агента лишь означает, что он работает нормально, но не разрешает ему автономно менять ваши файлы.
* **Агент не может симулировать решение человека.** Агент никогда не скажет "одобрено человеком" вместо вас. Решаете только вы.

## Как разрешать агенту сохранять изменения?
Используйте строгие команды, чтобы контролировать, когда агент сохраняет работу:
* Для коммита (сохранения в локальную историю): `AOS COMMIT OK`
* Для отправки на сервер (push): `AOS PUSH OK`
* Подробнее в [Командах авторизации](AUTHORIZATION-COMMANDS.md).
