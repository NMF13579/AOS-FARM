# Границы хранения (Storage Boundaries)

AOS-FARM строго разделяет, где хранятся правила, где документация, а где код продукта.

## Структура директорий
* `/aos/` = Ядро AOS-FARM. Здесь лежат правила, шаблоны и логика. Пользовательские данные сюда не пишутся.
* `/aos/root/` = Корневые шаблоны для будущего установщика.
* `/aos-modules/` = Контейнер для дополнительных AOS модулей.
* `/project/` = Рабочая директория для **документации** вашего проекта (ТЗ, черновики, Evidence).
* `/.aos-tmp/` = Только для временных файлов и логов (temporary scratch).
* **Исходный код продукта** = Остается в обычной структуре целевого репозитория, не внутри AOS-FARM папок.

## Критические правила
* **Пользовательские данные не хранятся в `/aos/`.**
* **Документация по проекту хранится в `/project/`.**
* **Исходный код по умолчанию не хранится в `/project/`.**
* **Временные файлы (`/.aos-tmp/`) не являются Source of Truth.**
* **Evidence (доказательства) — это не одобрение (approval).**
* **Отчеты — это не изменение статуса (lifecycle mutation).**
* **Папка `/.aos-tmp/` категорически не должна хранить:** Evidence, отчеты, подтверждения, чекпоинты, защищенные/канонические файлы или любые артефакты Source of Truth.
