# Установка AOS-FARM (Installation)

Это обзорный документ о том, как устанавливается AOS-FARM.

## Сухая проверка (Dry-Run)
*Реализовано (MVP)*

Перед тем как применять установку, необходимо выполнить "сухую проверку". Это покажет, что именно будет добавлено в ваш проект, без реального изменения файлов.

Команда для запуска:
`python3 aos/scripts/aos_install.py --dry-run`

**Важное замечание по структуре папок:** Перед тем как в будущем применять какие-либо изменения установщиком, убедитесь, что вы понимаете [Модель границ рабочего пространства (Workspace Boundary)](WORKSPACE-BOUNDARY.md). Установщик не будет работать с папкой `/project/` как с кодом продукта, а временная директория `/.aos-tmp/` никогда не должна использоваться для важных отчетов.

Dry-run:
* Находит корень репозитория и `/aos/root/`.
* Строит план разворачивания (mapping: `/aos/root/<relative-path>` -> `/<relative-path>`).
* Обнаруживает конфликты (если целевой файл уже существует, например `README.md`, `llms.txt`, `AGENTS.md`).
* Конфликты обрабатываются как `HUMAN_REVIEW_REQUIRED` (не перезаписываются молча).
* Запрещает создание директорий с исходным кодом внутри `/project/` (например, `src`, `tests`).
* Запрещает пути, выходящие за пределы корня или указывающие в `.git/` / `/.aos-tmp/`.
* **Ничего не изменяет в файловой системе.**
* Возвращает статусы: `PASS` / `PASS_WITH_WARNINGS` / `HUMAN_REVIEW_REQUIRED` / `BLOCKED` / `UNKNOWN_BLOCKED`.

**ВАЖНО:** Dry-run `PASS` не является approval.

## Установка (Apply)
*Статус: NOT_IMPLEMENTED*

Команда для применения установки:
`python3 aos/scripts/aos_install.py --apply`

Эта команда в данный момент заблокирована (возвращает `NOT_IMPLEMENTED`), так как этап сборки 580 реализует только MVP установщика в режиме dry-run.
