# Установка AOS-FARM (Installation)

Это обзорный документ о том, как устанавливается AOS-FARM.

## Сухая проверка (Dry-Run)
*Доступна в будущем*

Перед тем как применять установку, необходимо выполнить "сухую проверку". Это покажет, что именно будет добавлено в ваш проект, без реального изменения файлов.

Команда для запуска dry-run (когда будет реализована):
`python3 aos/scripts/aos_install.py --dry-run`

Dry-run обязан:
* проинспектировать `/aos/root/`;
* построить план установки;
* обнаружить потенциальные конфликты (например, если файл уже существует);
* обнаружить конфликт `README.md`;
* **никогда** не перезаписывать молча целевые файлы;
* **никогда** не писать пользовательские данные в `/aos/`;
* **никогда** не создавать файлы с исходным кодом продукта в `/project/`;
* возвращать статус: `PASS` / `PASS_WITH_WARNINGS` / `BLOCKED` / `UNKNOWN_BLOCKED` / `HUMAN_REVIEW_REQUIRED`.

## Установка (Apply)
*Статус: NOT_IMPLEMENTED (Запланировано)*

Команда для применения установки (когда будет реализована):
`python3 aos/scripts/aos_install.py --apply`

Эта команда будет переносить файлы из `/aos/root/` в корень вашего проекта, строго соблюдая ограничения безопасности, выявленные во время `dry-run`.
