# Команды авторизации (Authorization Commands)

Команды авторизации — это точные фразы, которые вы (человек) должны отправить агенту для одобрения действий. Агент не имеет права принимать ваши намерения за команды, если они не соответствуют строгому формату.

## Строгие команды

### AOS COMMIT OK
Значение:
* разрешает сделать ровно **один коммит**;
* применяется только для уже проверенного текущего изменения (diff) и строго в рамках текущей задачи (scope);
* **не** разрешает отправку на сервер (push);
* **не** разрешает релиз (release);
* **не** разрешает дополнительные правки вне scope;
* **не** разрешает расширение scope.

### AOS PUSH OK
Значение:
* разрешает сделать ровно **один push**;
* применяется только для уже созданного коммита и указанной удаленной ветки (remote branch);
* **не** разрешает создавать новые коммиты;
* **не** разрешает релиз;
* **не** разрешает дополнительные правки;
* **не** разрешает расширение scope.

### AOS MICRO COMMIT+PUSH OK
Значение:
* разрешает сделать ровно **один коммит и один push** сразу;
* применяется только для уже проверенных микро-исправлений (micro-correction diff/scope);
* **не** применимо для семантических изменений защищенных или канонических файлов (protected/canonical semantic changes);
* **не** применимо для изменений статуса жизненного цикла (lifecycle semantics);
* **не** применимо для изменений логики одобрения (approval semantics);
* **не** применимо для логики установщика (installer apply logic) или обновления (update apply logic);
* **не** применимо для изменений поведения валидаторов (validator behavior changes);
* **не** применимо для деструктивных операций;
* **не** применимо для автоматизации релизов (release automation).

## Правило обычных слов (Shortcut Policy)

Следующие слова считаются **только намерением или просьбой**, и **не являются авторизацией**:
* комит
* коммит
* commit
* пуш
* push
* залей
* отправь

Если вы напишете одно из этих слов, агент **не должен** выполнять commit/push. Вместо этого агент обязан показать точную команду, которую нужно скопировать и вставить.

* Для "комит" / "коммит" / "commit", агент покажет: `AOS COMMIT OK`
* Для "пуш" / "push" / "залей" / "отправь", агент покажет: `AOS PUSH OK`

## Критические напоминания
* Разрешение на коммит (Commit authorization) **не является** разрешением на пуш (push authorization).
* Разрешение на пуш (Push authorization) **не является** разрешением на релиз (release authorization).
* Успешный статус (PASS) **не является** одобрением (approval).
* Доказательства (Evidence) **не являются** одобрением (approval).
* Успешное прохождение CI (CI PASS) **не является** одобрением.
* Одобрение человека (Human approval) **не может быть симулировано** агентом.
