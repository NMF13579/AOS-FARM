# AOS-FARM Core Feature Documentation Registry

## Purpose
This directory (`docs/features/`) contains the manual feature documentation registry for the AOS-FARM project. It is designed to help non-programmers and vibe-coders understand what features exist, why they exist, and how to use them without relying on automated source code scanning or RAG.

## Policy
- **Manual Maintenance**: Features are documented manually. This registry is NOT an automated scanner.
- **Not an Approval Authority**: This registry does not authorize executions or mutations.
- **Not Source of Truth**: This registry does not override canonical project sources (`00_AOS_Core_Control.md`, `01`, `02`).
- **Not Evidence**: The registry itself is not an evidence artifact.

## What Belongs Here
- Explanations of completed and implemented core features.
- Known limitations, safety notes, and user workflows for these features.
- Links to relevant files and templates.

## What Does NOT Belong Here
- Automatically generated code indexes.
- RAG databases or embeddings.
- In-progress feature drafts or spec parking notes.

## How to Request a Documentation Update
If a new feature is implemented, use the `templates/feature-documentation-update-request-template.md` to ask the agent to document the feature.
