# Словарь терминов AOS-FARM (Глоссарий)

Добро пожаловать в глоссарий AOS-FARM! Этот документ простыми словами объясняет ключевые термины, чтобы вы могли уверенно и безопасно работать с системой.

## Основные понятия

* **AOS-FARM**: Безопасная "песочница" (репозиторий) для создания, тестирования и контроля работы автономных ИИ-агентов. AOS-FARM сейчас не является готовым к production (не production-ready).
* **AgentOS Next**: Будущая концепция операционной системы на базе агентов, для которой AOS-FARM служит тестовым полигоном.
* **Core Loop Ready**: Состояние репозитория, при котором он готов к выполнению базового цикла задач обычными пользователями.
* **First Consumer Workflow**: Шаблон и руководство, описывающие, как обычный пользователь (первый потребитель) может безопасно ставить задачи ИИ-агенту.
* **controlled task (контролируемая задача)**: Задача, которая ограничена строгими правилами, не может менять ключевые настройки без спроса и требует проверки человеком перед сохранением.
* **Task Brief (задание)**: Текстовый документ-шаблон, в котором пользователь описывает задачу для ИИ, указывает разрешенные файлы и запрашивает оценку рисков.
* **Source of Truth (Единый источник истины)**: Текущее состояние файлов в репозитории (особенно чекпойнтов и отчетов). Память ИИ не является источником истины — только сохраненные файлы.
* **canonical source (канонический источник)**: Базовые файлы правил (00, 01, 02), которые описывают, как работает система.
* **protected file (защищенный файл)**: Файл, который ИИ-агент не имеет права изменять без явного и отдельного разрешения от человека (например, файлы `00`, `01`, `02`).

## Авторизация и проверки человеком

* **Human Checkpoint (Чекпойнт человека)**: Специальный Markdown-файл, в котором человек должен вручную проставить решение (например, `APPROVED_FOR_COMMIT`), чтобы ИИ мог продолжить работу. 
* **Human Authorization (Авторизация человеком)**: Явное разрешение от человека на выполнение определенного действия (коммит, пуш). Авторизацию человека нельзя симулировать или пропустить.
* **approval (одобрение)**: Решение человека разрешить действие. Важно: технический "PASS" (успех) тестов не означает одобрение человека.
* **execution authorization (авторизация выполнения)**: Разрешение ИИ-агенту начать писать код или менять файлы.
* **commit authorization (авторизация коммита)**: Разрешение сохранить (сделать commit) измененные файлы в локальную историю Git.
* **push authorization (авторизация пуша)**: Разрешение отправить (push) сохраненные изменения на удаленный сервер (например, GitHub).
* **release authorization (авторизация релиза)**: Разрешение на создание новой публичной версии (релиза).
* **production use (использование в продакшене)**: Применение кода в реальных, рабочих, коммерческих условиях. Использование в production строго запрещено без отдельного разрешения. В AOS-FARM нет статуса готовности к production.

## Статусы и результаты тестов

* В AOS-FARM действуют строгие правила безопасности (Minimal Safety Floor):
  * **PASS (Успешно)**: Означает только то, что скрипт отработал без ошибок. Это **НЕ** является одобрением (approval) человека.
  * **Evidence (Доказательство)**: Отчеты или логи. Наличие доказательства **НЕ** является одобрением.
  * **CI PASS**: Успешное прохождение автоматических тестов. Это **НЕ** является одобрением для публикации или релиза.
  * **UNKNOWN (Неизвестно)**: Если статус неизвестен, он **НЕ** считается нормальным (OK). Система блокируется.
  * **NOT_RUN (Не запускалось)**: Если тест не запускался, он **НЕ** приравнивается к успешному (PASS).
  * **BLOCKED (Заблокировано)**: Состояние, при котором система останавливает любую работу из-за нарушения правил или непредвиденного состояния.
  * **HUMAN_REVIEW_REQUIRED (Требуется проверка человеком)**: Ситуация, в которой ИИ-агент не уверен в безопасности действий и останавливается, требуя вмешательства человека.

## Уровни риска (Risk Profiles)

* **Risk Profile (Профиль риска)**: Оценка того, насколько опасной является задача. ИИ-агент может предложить профиль риска, но **не может** назначить статус `LOW_RISK_FAST` сам себе.
* **LOW_RISK_FAST (Низкий риск, быстро)**: Задачи, которые почти ни на что не влияют (например, исправление опечаток). Разрешено выполнять быстрее.
* **MEDIUM_RISK_GUIDED (Средний риск, с управлением)**: Требует пошагового контроля и чекпойнтов. Используется по умолчанию.
* **HIGH_RISK_PROTECTED (Высокий риск, защищено)**: Задачи, меняющие ключевую архитектуру или канонические файлы. Требуют глубокого аудита.

## Термины Git и Разработки

* **remote baseline (удаленный бейзлайн)**: Текущее состояние кода на центральном сервере.
* **origin/dev**: Удаленная ветка `dev` на сервере (например, на GitHub).
* **tag (тег)**: Метка, фиксирующая определенную версию кода (например, `v5.4-final-min`).
* **GitHub release (релиз GitHub)**: Официальная публикация версии на GitHub для пользователей. На текущий момент GitHub-релизов в AOS-FARM нет.
* **destructive operation (деструктивная операция)**: Удаление файлов, истории, баз данных и т.д. Такие операции запрещены по умолчанию.
* **runner**: Автономный исполнитель кода (автоматически выполняющий задачи). В AOS-FARM автономный runner отключен и его использование сейчас не является безопасным.
* **dogfood (догфудинг)**: Проверка собственных инструкций или продукта на самих себе (как бы "поедание собственного собачьего корма").
* **dry-run (холостой прогон)**: Тестовый запуск процесса без реального внесения изменений, только для проверки логики.
* **skeleton (скелет/каркас)**: Пустые файлы или базовая структура без реального кода. Скелет **НЕ** является реальной реализацией (implementation).
* **implementation (реализация)**: Написание настоящего, работающего кода для выполнения задачи.
