# Agentic Operating System (AOS)

**Безопасный фреймворк для разработки ПО с помощью ИИ-агентов.**

AOS — это методологический фреймворк, основанный на Markdown, созданный для безопасного управления автономными ИИ-агентами. Вместо того чтобы полагаться на активные Python-раннеры, CI-хуки или базы данных, AOS обеспечивает строгие границы безопасности (fail-closed) с помощью детерминированного управления файлами и явных человеческих чекпоинтов.

## Что такое AOS?
AOS поставляется в виде автономного потребительского набора (consumer kit), расположенного в директории `aos/`. Он предоставляет:
- **Строгую методологию**: Правила, которым должны подчиняться ИИ-агенты (например, `PASS ≠ approval`).
- **Шаблоны**: Чекпоинты, описания задач и отчёты об исполнении.
- **Промпты**: Стандартизированные стартовые инструкции для агентов.
- **Управление**: Чёткие границы, определяющие, что агент может и чего не может делать автономно.

## Основная философия
1. **Fail-Closed по умолчанию**: Если состояние неизвестно, агент обязан остановиться и запросить помощь.
2. **PASS ≠ Approval**: Успешное прохождение теста не даёт права на коммит или деплой.
3. **Запрет на симуляцию одобрения**: Агентам строго запрещено писать "Approved by Human" без реального вмешательства человека.
4. **Отложенные коммиты**: Агенты выполняют работу локально и ожидают явного разрешения на пакетный коммит.

## Быстрый старт
Чтобы внедрить AOS в ваш проект:
1. Скопируйте папку `aos/` в корень вашего проекта.
2. Скопируйте `aos/root/AGENTS.md` в корень вашего проекта.
3. Дайте инструкцию вашему ИИ-агенту прочитать `aos/START_HERE.md`. Это единственный обязательный стартовый документ, который проведет вас через Problem Intake и Technical Assignment до написания какого-либо кода.

## Никакой магии
Для работы AOS не требуются активные Python-раннеры, скрипты непрерывной интеграции (CI), базы данных или компоненты RAG. Фреймворк полностью опирается на способность ИИ-агента следовать чётким системным правилам, описанным в Markdown.
