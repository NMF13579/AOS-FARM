# Project Brief Template

## Метаданные

```yaml
document_type: project_brief
project: [Название проекта]
status: DRAFT
owner: [Имя / Роль]
```

## 1. Проблема

[Какую конкретно проблему решает данный проект? Почему это важно?]

## 2. Целевая аудитория / Пользователь

[Для кого создается проект? Кто конечный потребитель?]

## 3. Цель (Goal)

[Какой конечный результат ожидается? Измеримый критерий успеха.]

## 4. Ограничения (Constraints)

[Временные, технические, бизнес-ограничения. Что мы точно НЕ делаем (Non-goals)?]

## 5. Допущения (Assumptions)

[Что мы считаем истинным на старте проекта, но что требует проверки?]

## Ограничение ответственности

Этот документ определяет границы проекта, но **не является разрешением на исполнение кода** (execution approval). Для перехода к разработке требуется Specification и Task Brief, а также явное назначение Risk Profile человеком.
